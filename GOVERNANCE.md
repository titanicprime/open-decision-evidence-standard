# Governance

## Current status

This repository is maintained as a public discussion draft for the Open Decision Evidence Standard. The current draft version is v0.1 and should be treated as a candidate standard under active review rather than a settled specification.

## Maintainer role

Cognous currently acts as the initial maintainer. The maintainer role is administrative and editorial: curate issues, review pull requests, publish draft updates, and keep the repository usable for public comment.

## Standards neutrality goal

The stated goal is neutral standards governance if the work attracts meaningful external participation. The format should remain open, vendor-neutral, and freely implementable regardless of whether commercial implementations emerge around it.

## Decision process for v0.x drafts

For v0.x drafts, changes may be proposed through issues and pull requests. The maintainer will consider technical correctness, interoperability, privacy, security, portability, and the impact on open implementation.

Material changes should be documented in RFCs or issue discussions before inclusion in a numbered draft.

## Future neutral governance options

If the effort gains adoption, future governance options may include transfer to a neutral foundation, stewardship by a multi-party working group, or another open standards process with public participation and transparent decision records.

## Versioning

Draft versions use `v0.x` numbering to signal active design iteration. A future `v1.0` should require clearer governance, wider review, and more stable conformance expectations.

## Conformance profiles

Profiles may evolve independently from the base schema when needed for sector-specific or operational needs. The base specification should remain smaller than any one implementation profile.

## Conflict-of-interest principle

Technical decisions should be evaluated on whether they improve openness, portability, verifiability, and implementation neutrality. Maintainers and contributors should avoid using the specification process to privilege proprietary dependencies or vendor-specific control mechanisms.

## Commercial implementation separation

Commercial software may implement, verify, route, or govern records defined by this repository, but commercial implementation architecture is outside the scope of the standard. Repository content should describe the open record format, not proprietary platform design.
